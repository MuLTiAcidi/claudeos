# Report Generator Agent

Auto-generates system, security, and performance reports for Linux servers. Collects stats from `df`, `free`, `top`, `vmstat`, `iostat`, renders them into Markdown/HTML templates, charts metrics with `gnuplot`, converts to PDF via `pandoc`, and delivers via email or webhook on a cron schedule.

---

## Safety Rules

- NEVER include secrets, API keys, or passwords in generated reports.
- ALWAYS write reports to `/var/lib/reports/` (chmod 750, owned by report user).
- NEVER email reports to addresses not in the allow-list.
- ALWAYS sanitize log excerpts before embedding (strip tokens, IPs if required).
- NEVER run report scripts as root unless collecting privileged data — use a dedicated `reporter` user.
- ALWAYS rotate old reports — keep max 90 days unless archived elsewhere.
- Log every report generation to `/var/log/report-generator.log`.

## 1. Required Tools

### Install Reporting Toolchain

```bash
sudo apt update
sudo apt install -y sysstat procps coreutils gnuplot pandoc texlive-xetex \
    msmtp mailutils bsd-mailx jq curl
```

### Verify Tools

```bash
for t in df free top vmstat iostat mpstat sar gnuplot pandoc mail curl jq; do
    command -v "$t" >/dev/null 2>&1 && echo "OK: $t" || echo "MISSING: $t"
done
```

### Enable sysstat Collection (for sar/iostat history)

```bash
sudo sed -i 's/^ENABLED="false"/ENABLED="true"/' /etc/default/sysstat
sudo systemctl enable --now sysstat
```

## 2. System Stats Collection

### CPU Snapshot

```bash
top -bn1 | head -20
```

### CPU Load Average

```bash
uptime | awk -F'load average:' '{print $2}'
```

### Memory Usage

```bash
free -h
free -m | awk 'NR==2{printf "Used: %s/%s MB (%.1f%%)\n", $3,$2,$3*100/$2}'
```

### Disk Usage

```bash
df -hT --output=source,fstype,size,used,avail,pcent,target | grep -vE '^tmpfs|^devtmpfs'
```

### Disk I/O

```bash
iostat -xk 1 3 | tail -n +4
```

### Virtual Memory

```bash
vmstat 1 5
```

### Network Interfaces

```bash
ip -s link show | grep -A1 -E '^[0-9]+:'
```

### Top Processes by CPU

```bash
ps -eo pid,user,%cpu,%mem,comm --sort=-%cpu | head -11
```

### Top Processes by Memory

```bash
ps -eo pid,user,%cpu,%mem,comm --sort=-%mem | head -11
```

### Open Files / Sockets

```bash
ss -s
lsof | wc -l
```

### Uptime

```bash
uptime -p
```

### System Info Block

```bash
hostnamectl
cat /etc/os-release
```

## 5. Markdown Report Template

### Save as `/usr/local/share/reports/templates/system-daily.md.tpl`

```markdown
# System Report — ${HOSTNAME}

**Generated:** ${REPORT_DATE}
**Period:** ${REPORT_PERIOD}
**Operator:** ${REPORT_USER}

---

## Overview

- **Hostname:** ${HOSTNAME}
- **OS:** ${OS_NAME}
- **Kernel:** ${KERNEL}
- **Uptime:** ${UPTIME}
- **Load Average:** ${LOADAVG}

## CPU

\`\`\`
${CPU_BLOCK}
\`\`\`

## Memory

\`\`\`
${MEM_BLOCK}
\`\`\`

## Disk Usage

\`\`\`
${DISK_BLOCK}
\`\`\`

## Top Processes (CPU)

\`\`\`
${TOP_CPU}
\`\`\`

## Top Processes (Memory)

\`\`\`
${TOP_MEM}
\`\`\`

## Security Summary

- Failed SSH attempts (24h): **${FAILED_SSH}**
- Active fail2ban bans: **${BANNED_IPS}**
- Pending security updates: **${SEC_UPDATES}**

## Charts

![CPU Usage](cpu-chart.png)
![Memory Usage](mem-chart.png)

---

*Generated by ClaudeOS Report Generator*
```

### Render Template with envsubst

```bash
export HOSTNAME="$(hostname)"
export REPORT_DATE="$(date '+%Y-%m-%d %H:%M:%S')"
export REPORT_PERIOD="last 24 hours"
export REPORT_USER="$(whoami)"
export OS_NAME="$(. /etc/os-release && echo "$PRETTY_NAME")"
export KERNEL="$(uname -r)"
export UPTIME="$(uptime -p)"
export LOADAVG="$(uptime | awk -F'load average:' '{print $2}')"
export CPU_BLOCK="$(top -bn1 | head -10)"
export MEM_BLOCK="$(free -h)"
export DISK_BLOCK="$(df -hT | grep -vE '^tmpfs|^devtmpfs')"
export TOP_CPU="$(ps -eo pid,user,%cpu,%mem,comm --sort=-%cpu | head -11)"
export TOP_MEM="$(ps -eo pid,user,%cpu,%mem,comm --sort=-%mem | head -11)"
export FAILED_SSH="$(journalctl _SYSTEMD_UNIT=ssh.service --since '24 hours ago' | grep -c 'Failed password')"
export BANNED_IPS="$(sudo fail2ban-client status sshd 2>/dev/null | grep -oP 'Currently banned:\s+\K\d+' || echo 0)"
export SEC_UPDATES="$(apt list --upgradable 2>/dev/null | grep -ic security)"

envsubst < /usr/local/share/reports/templates/system-daily.md.tpl > /var/lib/reports/system-$(date +%F).md
```

## 7. Charts with gnuplot

### Collect CPU Samples to a Data File

```bash
DATAFILE=/tmp/cpu-samples.dat
> "$DATAFILE"
for i in $(seq 1 60); do
    USAGE=$(top -bn1 | awk '/^%Cpu/ {print 100-$8}')
    echo "$i $USAGE" >> "$DATAFILE"
    sleep 1
done
```

### gnuplot CPU Chart Script

```bash
cat > /tmp/cpu-chart.gp <<'EOF'
set terminal pngcairo size 800,400 enhanced font 'Arial,11'
set output '/var/lib/reports/cpu-chart.png'
set title "CPU Usage (last 60 seconds)"
set xlabel "Time (s)"
set ylabel "CPU %"
set yrange [0:100]
set grid
set style line 1 lc rgb '#0060ad' lt 1 lw 2 pt 7 ps 0.5
plot "/tmp/cpu-samples.dat" using 1:2 with linespoints ls 1 title "CPU"
EOF
gnuplot /tmp/cpu-chart.gp
```

### Memory Chart from `free` Samples

```bash
> /tmp/mem-samples.dat
for i in $(seq 1 60); do
    USED=$(free -m | awk 'NR==2{printf "%.1f", $3*100/$2}')
    echo "$i $USED" >> /tmp/mem-samples.dat
    sleep 1
done

cat > /tmp/mem-chart.gp <<'EOF'
set terminal pngcairo size 800,400
set output '/var/lib/reports/mem-chart.png'
set title "Memory Usage"
set xlabel "Sample"
set ylabel "Used %"
set yrange [0:100]
set grid
plot "/tmp/mem-samples.dat" using 1:2 with filledcurves x1 lc rgb '#a06030' title "Memory"
EOF
gnuplot /tmp/mem-chart.gp
```

### Disk Bar Chart

```bash
df -h --output=target,pcent | tail -n +2 \
    | awk '{gsub("%","",$2); print NR, $2, $1}' > /tmp/disk.dat

cat > /tmp/disk.gp <<'EOF'
set terminal pngcairo size 800,400
set output '/var/lib/reports/disk-chart.png'
set title "Disk Usage by Mountpoint"
set ylabel "% Used"
set yrange [0:100]
set style data histograms
set style fill solid 0.7
set boxwidth 0.6
set xtics rotate by -30
plot "/tmp/disk.dat" using 2:xtic(3) title "Used %"
EOF
gnuplot /tmp/disk.gp
```

## 8. PDF Generation with pandoc

### Markdown → PDF

```bash
pandoc /var/lib/reports/system-$(date +%F).md \
    -o /var/lib/reports/system-$(date +%F).pdf \
    --pdf-engine=xelatex \
    -V geometry:margin=1in \
```

```bash
sudo apt install -y wkhtmltopdf

    /var/lib/reports/system-$(date +%F).pdf
```

### Markdown → PDF with Charts Embedded

```bash
cd /var/lib/reports
pandoc system-$(date +%F).md \
    -o system-$(date +%F).pdf \
    --pdf-engine=xelatex \
    --resource-path=/var/lib/reports \
    -V geometry:margin=0.75in \
    -V colorlinks=true
```

### Markdown → DOCX (alternative format)

```bash
pandoc /var/lib/reports/system-$(date +%F).md \
    -o /var/lib/reports/system-$(date +%F).docx
```

## 9. Email Delivery

### Configure msmtp for Outbound Mail

```bash
sudo apt install -y msmtp msmtp-mta
sudo tee /etc/msmtprc <<'EOF'
defaults
auth on
tls on
tls_trust_file /etc/ssl/certs/ca-certificates.crt
logfile /var/log/msmtp.log

account default
host smtp.example.com
port 587
from reports@example.com
user reports@example.com
password CHANGE_ME
EOF
sudo chmod 600 /etc/msmtprc
```

### Send Report by Email (mail command)

```bash
echo "Daily system report attached." | mail \
    -s "System Report — $(hostname) — $(date +%F)" \
    -A /var/lib/reports/system-$(date +%F).pdf \
    ops@example.com
```

### Send via msmtp Directly

```bash
{
    echo "Subject: System Report $(hostname) $(date +%F)"
    echo "From: reports@example.com"
    echo "To: ops@example.com"
    echo "Content-Type: text/html"
    echo ""
    cat /var/lib/reports/system-$(date +%F).html
} | msmtp ops@example.com
```

### Send with sendmail and MIME Attachment

```bash
BOUNDARY="==BOUNDARY=="
{
    echo "From: reports@example.com"
    echo "To: ops@example.com"
    echo "Subject: System Report $(date +%F)"
    echo "MIME-Version: 1.0"
    echo "Content-Type: multipart/mixed; boundary=\"$BOUNDARY\""
    echo ""
    echo "--$BOUNDARY"
    echo "Content-Type: text/plain"
    echo ""
    echo "See attached PDF report."
    echo ""
    echo "--$BOUNDARY"
    echo "Content-Type: application/pdf"
    echo "Content-Transfer-Encoding: base64"
    echo "Content-Disposition: attachment; filename=\"report.pdf\""
    echo ""
    base64 /var/lib/reports/system-$(date +%F).pdf
    echo "--$BOUNDARY--"
} | sendmail -t
```

## 10. Full Daily Report Script

### Save as `/usr/local/bin/generate-daily-report.sh`

```bash
#!/bin/bash
set -euo pipefail

REPORT_DIR=/var/lib/reports
TEMPLATE=/usr/local/share/reports/templates/system-daily.md.tpl
LOG=/var/log/report-generator.log
DATE=$(date +%F)
OUT_MD="$REPORT_DIR/system-$DATE.md"
OUT_PDF="$REPORT_DIR/system-$DATE.pdf"

mkdir -p "$REPORT_DIR"

log() { echo "[$(date '+%F %T')] $*" | tee -a "$LOG"; }

log "Starting daily report for $(hostname)"

export HOSTNAME="$(hostname)"
export REPORT_DATE="$(date '+%F %T')"
export REPORT_PERIOD="last 24 hours"
export REPORT_USER="$(whoami)"
export OS_NAME="$(. /etc/os-release && echo "$PRETTY_NAME")"
export KERNEL="$(uname -r)"
export UPTIME="$(uptime -p)"
export LOADAVG="$(uptime | awk -F'load average:' '{print $2}')"
export CPU_BLOCK="$(top -bn1 | head -10)"
export MEM_BLOCK="$(free -h)"
export DISK_BLOCK="$(df -hT | grep -vE '^tmpfs|^devtmpfs')"
export TOP_CPU="$(ps -eo pid,user,%cpu,%mem,comm --sort=-%cpu | head -11)"
export TOP_MEM="$(ps -eo pid,user,%cpu,%mem,comm --sort=-%mem | head -11)"
export FAILED_SSH="$(journalctl _SYSTEMD_UNIT=ssh.service --since '24 hours ago' 2>/dev/null | grep -c 'Failed password' || echo 0)"
export BANNED_IPS="$(sudo fail2ban-client status sshd 2>/dev/null | grep -oP 'Currently banned:\s+\K\d+' || echo 0)"
export SEC_UPDATES="$(apt list --upgradable 2>/dev/null | grep -ic security || echo 0)"

log "Rendering markdown template"
envsubst < "$TEMPLATE" > "$OUT_MD"

log "Generating charts"
/usr/local/bin/generate-charts.sh || log "Charts failed, continuing"

log "Converting to PDF"
pandoc "$OUT_MD" -o "$OUT_PDF" --pdf-engine=xelatex \
    --resource-path="$REPORT_DIR" \
    -V geometry:margin=0.75in 2>>"$LOG" || log "PDF conversion failed"

log "Emailing report"
echo "Daily system report for $(hostname) — $DATE" | mail \
    -s "System Report $(hostname) $DATE" \
    -A "$OUT_PDF" \
    ops@example.com 2>>"$LOG" || log "Email send failed"

log "Done. Report at $OUT_PDF"
```

### Make executable

```bash
sudo chmod +x /usr/local/bin/generate-daily-report.sh
```

## 11. Weekly and Monthly Reports

### Weekly Aggregation Script

```bash
#!/bin/bash
WEEK=$(date +%V)
YEAR=$(date +%Y)
OUT=/var/lib/reports/weekly-${YEAR}W${WEEK}.md

{
    echo "# Weekly Report — Week $WEEK, $YEAR"
    echo ""
    echo "## Disk Trend"
    sar -f /var/log/sysstat/sa$(date -d '7 days ago' +%d) 2>/dev/null || true
    echo ""
    echo "## Top SSH Sources (week)"
    journalctl --since "7 days ago" _SYSTEMD_UNIT=ssh.service \
        | grep "Failed password" \
        | grep -oP 'from \K[0-9.]+' \
        | sort | uniq -c | sort -rn | head -10
    echo ""
    echo "## Package Changes"
    grep " install \| upgrade \| remove " /var/log/dpkg.log \
        | tail -50
} > "$OUT"

pandoc "$OUT" -o "${OUT%.md}.pdf" --pdf-engine=xelatex
```

## 12. Cron Scheduling

### Daily Report at 06:00

```bash
sudo crontab -e
# Add:
0 6 * * * /usr/local/bin/generate-daily-report.sh >/dev/null 2>&1
```

### Weekly Report Monday 07:00

```bash
0 7 * * 1 /usr/local/bin/generate-weekly-report.sh >/dev/null 2>&1
```

### Monthly Report 1st of Month

```bash
0 8 1 * * /usr/local/bin/generate-monthly-report.sh >/dev/null 2>&1
```

### systemd Timer Alternative

```bash
sudo tee /etc/systemd/system/report-daily.service <<'EOF'
[Unit]
Description=Generate daily system report

[Service]
Type=oneshot
ExecStart=/usr/local/bin/generate-daily-report.sh
EOF

sudo tee /etc/systemd/system/report-daily.timer <<'EOF'
[Unit]
Description=Daily report timer

[Timer]
OnCalendar=*-*-* 06:00:00
Persistent=true

[Install]
WantedBy=timers.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now report-daily.timer
```

### Verify Timer

```bash
systemctl list-timers | grep report
```

## 16. Troubleshooting

### pandoc PDF Fails

```bash
sudo apt install -y texlive-xetex texlive-fonts-recommended texlive-latex-extra
pandoc test.md -o test.pdf --pdf-engine=xelatex 2>&1 | tail -20
```

### gnuplot Missing PNG Support

```bash
gnuplot -e "set terminal" 2>&1 | grep -i png
sudo apt install -y gnuplot-nox  # or gnuplot-x11
```

### Mail Not Sending

```bash
tail -30 /var/log/msmtp.log
echo "test" | msmtp -d ops@example.com
```

### Charts Missing in PDF

```bash
# Ensure resource-path points to chart directory
pandoc report.md -o report.pdf --resource-path=/var/lib/reports
```

## Output Format

When generating a report, always show:

1. **Hostname / date / period**
2. **Files generated** (paths and sizes)
3. **Delivery status** (email sent / webhook posted)
4. **Next scheduled run**
5. **Log line** appended to `/var/log/report-generator.log`
